Verify All SD-WAN Edge Configurations Are In-Sync

Test Status: FAILED

Description

This test validates configuration synchronization status for all managed SD-WAN edge devices
by querying the SD-WAN Manager REST API. SD-WAN edge devices must have a configured system IP and be in a managed
state to ensure that intended configurations are properly applied and running. Configuration synchronization is
critical for device stability, security policy enforcement, and consistent application performance across the SD-WAN fabric.

Setup

  • Access to an active SD-WAN Manager controller is available via HTTPS API.
  • Authentication credentials for the SD-WAN Manager controller are valid and configured.
  • The SD-WAN fabric is deployed with managed edge devices that have completed onboarding.
  • Edge devices have system IPs assigned and are not in an 'Unmanaged' state.

Procedure

  • Establish HTTPS connection to the SD-WAN Manager controller.
  • Query the SD-WAN Manager API endpoint: /dataservice/system/device/vedges.
  • Retrieve the JSON response containing all edge device objects.
  • For EACH device object:
    • INCLUDE device in validation scope IF configuredSystemIP is present AND managed-by != 'Unmanaged'.
    • For each included device, verify that configStatusMessage equals 'In Sync'.
    • Record devices with missing, empty, or non-'In Sync' configStatusMessage as unhealthy.
  • Summarize results, reporting total checked devices, healthy (in sync), and unhealthy (out-of-sync) devices.

Pass/Fail Criteria

This test passes when all of the following conditions are met:

  • At least one SD-WAN edge device is discovered with a configured system IP and is managed.
  • For all included devices, the configStatusMessage attribute is present and equals 'In Sync'.
  • No device within validation scope is missing the configStatusMessage attribute or shows 'Out-of-Sync' or empty values.
  • The API query completes successfully and returns a valid response.

This test fails if any of the following criteria are met:

  • No managed SD-WAN edge devices with a configured system IP are discovered in the API response.
  • ANY device in scope is missing the configStatusMessage attribute, has an empty string, or shows 'Out-of-Sync'.
  • The API query fails or returns an error response.
