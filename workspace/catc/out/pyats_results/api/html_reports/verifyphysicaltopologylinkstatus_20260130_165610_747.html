Verify Physical Topology Link Status

Test Status: PASSED
Jobfile: tests/pyats/verify_dnac_physical_topology_link_status.py

Description

This test validates the operational status of all physical topology links
discovered by Cisco Catalyst Center. Physical topology links represent direct physical
connections between network devices, providing essential connectivity for network traffic
and application flow.

Ensuring that all discovered links have status 'up' confirms the underlying infrastructure
is intact, cable connections are present, and interfaces are enabled and healthy. This
verification is critical for network resilience, device reachability, and overall
infrastructure stability.

Setup

  • Access to an active Cisco Catalyst Center controller via HTTPS API.
  • Authentication credentials for Catalyst Center are valid and configured.
  • Network devices have been discovered and mapped into the physical topology.
  • Device discovery and topology mapping have completed successfully.

Procedure

  • Establish HTTPS connection to the Catalyst Center controller.
  • Query the Catalyst Center API endpoint: /dna/intent/api/v1/topology/physical-topology.
  • Retrieve the physical topology JSON response containing all discovered links.
  • Parse the JSON response and extract the response.links[] array.
  • For EACH discovered link:
    • Extract source and target device UUIDs, start and end port names, and link ID.
    • Extract the linkStatus attribute for the link.
    • Verify that linkStatus equals 'up'.

Pass/Fail Criteria

This test passes when all of the following conditions are met:

  • At least one physical topology link is discovered in Catalyst Center.
  • ALL discovered physical topology links have linkStatus equal to 'up'.

This test fails if any of the following criteria are met:

  • No physical topology links are discovered in the API response.
  • ANY discovered link has linkStatus other than 'up' (e.g., 'down', empty, or missing).
  • The API query fails or returns an error response.
