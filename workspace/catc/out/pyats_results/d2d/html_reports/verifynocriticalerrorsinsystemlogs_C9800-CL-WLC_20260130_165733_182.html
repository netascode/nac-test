Description

This test checks the system log on an IOS-XE device for the presence of any
critical errors, hardware failures, configuration rollbacks, AAA failures, software exceptions,
or recurring error-level messages. System logs are essential for identifying operational issues,
hardware faults, authentication problems, or abnormal software behavior. Ensuring that the
system log is free of high-severity or persistent errors is vital for network reliability,
device stability, and early detection of infrastructure issues.

Setup

  • SSH access to the target IOS-XE network device is available.
  • Authentication credentials for the device are valid and configured.
  • System logging is enabled and log buffer contains current entries.
  • The device is operational and reachable via the management network.

Procedure

  • Establish SSH connection to the IOS-XE network device.
  • Execute the CLI command: show logging to retrieve the device system log buffer.
  • Parse the command output using the Genie parser to obtain structured log entries.
  • For EACH log entry discovered in the parsed output:
    • Extract the log message severity from the syslog header (e.g., %FAC-SEV-CODE).
    • Check for the following:
      • Any log entry with severity 0 (Emergency), 1 (Alert), or 2 (Critical).
      • Any message text indicating hardware failures (e.g., power, fan, module).
      • Any message text indicating configuration rollback or abort.
      • Any message text indicating AAA (authentication/authorization/accounting) failure.
      • Any message text indicating a software exception or crash.
      • Recurring (persistent) severity 3 (Error) messages by counting repeated signatures.
  • Summarize results, reporting any discovered critical or recurring issues.

Pass/Fail Criteria

This test passes when all of the following conditions are met:

  • At least one log entry is discovered in the system log buffer.
  • No log entries have severity 0, 1, or 2 (Emergency, Alert, Critical).
  • No log entries indicate hardware failures (e.g., FAN, PSU, module, overheat, sensor).
  • No log entries indicate configuration rollback or abort events.
  • No log entries indicate AAA (authentication/authorization/accounting) failures.
  • No log entries indicate software exceptions or crashes.
  • No recurring (persistent) severity 3 (Error) messages with the same signature.

This test fails if any of the following criteria are met:

  • The system log buffer is empty or cannot be parsed.
  • One or more log entries have severity 0, 1, or 2 (Emergency, Alert, Critical).
  • Any log entry indicates hardware failure, configuration rollback, AAA failure, or software exception.
  • One or more recurring severity 3 (Error) messages are found with the same signature.
  • The CLI command fails to execute or returns an error.
  • The Genie parser fails to parse the command output.
