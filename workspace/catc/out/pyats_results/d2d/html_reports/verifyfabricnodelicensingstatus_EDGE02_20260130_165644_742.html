Verify SDA Fabric Node Licensing Status (EDGE02)

Test Status: PASSED
Jobfile: tests/pyats/verify_iosxe_fabric_node_licensing_status.py

Description

This test ensures that all required SDA (Software-Defined Access) fabric node licenses are present and active on the device.
Specifically, it validates the existence, status, and attributes of 'network-advantage' and 'dna-advantage' licenses, which are essential for proper fabric operation.
Verifying license presence and validity prevents feature restrictions, ensures compliance, and supports seamless SDA provisioning and automation.

Setup

  • SSH access to the IOS-XE network device is available.
  • Authentication credentials are valid and configured on the device.
  • Device is registered for smart licensing and has internet connectivity if required.
  • The device is deployed as part of an SDA fabric or expected to support SDA features.

Procedure

  • Establish SSH connection to the target network device.
  • Execute the CLI command: show license all.
  • Parse the output using the Genie parser if available, or fall back to regex parsing if necessary.
  • For EACH discovered license item (from parsed output or regex):
    • Identify all licenses where the name matches 'network-advantage' or 'dna-advantage' (case-insensitive, spaces or dashes allowed).
    • Extract and record the following attributes:
      • License name
      • License type (Perpetual or Subscription)
      • Status (IN USE/NOT IN USE)
      • Count
      • Enforcement type
      • Export status
  • For each license type ('network-advantage' and 'dna-advantage'), verify the following:
    • Status is 'IN USE'
    • License type: 'network-advantage' must be 'Perpetual'; 'dna-advantage' can be 'Perpetual' or 'Subscription'
    • Count is greater than or equal to 1
    • Export status is 'NOT RESTRICTED'
  • Summarize results and report any missing, unhealthy, or incorrectly attributed licenses.

Pass/Fail Criteria

This test passes when all of the following conditions are met:

  • At least one 'network-advantage' license is present with:
    • Status 'IN USE'
    • License type 'Perpetual'
    • Count >= 1
    • Export status 'NOT RESTRICTED'
  • At least one 'dna-advantage' license is present with:
    • Status 'IN USE'
    • License type 'Perpetual' or 'Subscription'
    • Count >= 1
    • Export status 'NOT RESTRICTED'
  • CLI command output is successfully parsed (either Genie or regex) and all license checks are performed without error.

This test fails if any of the following criteria are met:

  • No 'network-advantage' or 'dna-advantage' license is discovered in the device output.
  • Either license is present but does not have status 'IN USE'.
  • 'network-advantage' license type is not 'Perpetual'.
  • 'dna-advantage' license type is not 'Perpetual' or 'Subscription'.
  • Count < 1 for either license.
  • Export status is not 'NOT RESTRICTED' for either license.
  • CLI command execution or parsing fails, or no license information can be extracted.
